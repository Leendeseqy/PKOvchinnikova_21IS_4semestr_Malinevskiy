# Бизнес-требования к системе "Планировщик индивидуального образовательного маршрута"

## 1. Введение

### 1.1. Назначение документа
Документ описывает бизнес-требования к программному обеспечению "Планировщик индивидуального образовательного маршрута" для студентов высших учебных заведений.

### 1.2. Область действия
Система предназначена для планирования и отслеживания индивидуального образовательного пути студентов, управления компетенциями и навыками, а также формирования отчетности.

### 1.3. Цели проекта
1. Создать единую платформу для управления образовательными целями студентов
2. Обеспечить мониторинг развития компетенций и навыков
3. Автоматизировать процесс формирования отчетов об образовательном прогрессе
4. Стимулировать студентов через систему достижений
5. Обеспечить интеграцию с существующими образовательными стандартами

### 1.4. Целевая аудитория
- Студенты высших учебных заведений
- Кураторы учебных групп
- Методисты и преподаватели
- Администрация образовательных учреждений

## 2. Общее описание

### 2.1. Обзор продукта
Система представляет собой desktop-приложение с графическим интерфейсом для планирования и контроля индивидуального образовательного маршрута студента.

### 2.2. Бизнес-контекст
В современных условиях высшего образования требуется персонализированный подход к обучению. Система помогает студентам:
- Самостоятельно планировать образовательные цели
- Отслеживать прогресс в развитии компетенций
- Получать рекомендации для профессионального роста
- Формировать портфолио достижений

### 2.3. Основные возможности
- Управление образовательными целями различных типов
- Мониторинг развития навыков и компетенций
- Геймификация через систему достижений
- Формирование структурированных отчетов
- Статистический анализ образовательного прогресса

### 2.4. Ограничения
- Система ориентирована на индивидуальное использование студентом
- Требует базовых компьютерных навыков
- Имеет ограниченные возможности интеграции с внешними системами ВУЗа

## 3. Функциональные требования

### 3.1. Модуль управления целями
**FR-01:** Система должна позволять создавать образовательные цели
- **FR-01.01:** Возможность указания названия цели
- **FR-01.02:** Выбор типа цели из предопределенного списка (Курс, Проект, Самостоятельная работа и т.д.)
- **FR-01.03:** Установка статуса цели (Планируется, В процессе, Завершено, Отменено)
- **FR-01.04:** Задание плановой и фактической даты выполнения
- **FR-01.05:** Добавление подробного описания с поддержкой Markdown
- **FR-01.06:** Связывание цели с навыками (до 3-х навыков на цель)
- **FR-01.07:** Связывание цели с компетенциями и указание уровня развития

**FR-02:** Система должна предоставлять функции управления целями
- **FR-02.01:** Редактирование существующих целей
- **FR-02.02:** Удаление целей с подтверждением
- **FR-02.03:** Фильтрация и сортировка целей по различным критериям
- **FR-02.04:** Поиск целей по названию и описанию

### 3.2. Модуль управления навыками и компетенциями
**FR-03:** Система должна управлять базой навыков
- **FR-03.01:** Автоматическое предложение навыков при вводе
- **FR-03.02:** Статистика использования навыков в целях
- **FR-03.03:** Отображение наиболее развиваемых навыков

**FR-04:** Система должна управлять компетенциями
- **FR-04.01:** Загрузка компетенций из JSON-файла
- **FR-04.02:** Группировка компетенций по категориям
- **FR-04.03:** Расчет среднего уровня развития компетенции
- **FR-04.04:** Выявление слабых зон (компетенции с низким уровнем развития)
- **FR-04.05:** Формирование рекомендаций по развитию компетенций

### 3.3. Модуль достижений
**FR-05:** Система должна реализовывать геймификацию
- **FR-05.01:** Автоматическая проверка условий для получения достижений
- **FR-05.02:** Отображение статуса достижений (получено/не получено)
- **FR-05.03:** Система из 5 предопределенных достижений:
  - "Старт" - за создание первой цели
  - "Пунктуальный" - за завершение 3+ целей в срок
  - "Многоцелевой" - за наличие целей 3+ разных типов
  - "Навыковый рост" - за 4+ завершенные цели с одним навыком
  - "Планировщик" - за 5+ одновременно активных целей

### 3.4. Модуль отчетности
**FR-06:** Система должна генерировать отчеты
- **FR-06.01:** Формирование структурированного отчета в формате DOCX
- **FR-06.02:** Включение в отчет следующих разделов:
  - Общая информация
  - Список учебных целей
  - Развиваемые навыки
  - Уровень компетенций
  - Области для развития
  - Полученные достижения
  - Цели на семестр
  - Рекомендации
- **FR-06.03:** Автоматическое форматирование отчета согласно требованиям
- **FR-06.04:** Поддержка гиперссылок в описаниях целей

**FR-07:** Система должна предоставлять статистику
- **FR-07.01:** Статистика по типам целей
- **FR-07.02:** Процент целей, завершенных в срок
- **FR-07.03:** Динамика развития навыков
- **FR-07.04:** Прогресс по целям на семестр

### 3.5. Модуль целей на семестр
**FR-08:** Система должна поддерживать планирование на семестр
- **FR-08.01:** Создание семестровых целей с указанием типа
- **FR-08.02:** Отслеживание прогресса выполнения
- **FR-08.03:** Автоматическое обновление прогресса на основе завершенных целей
- **FR-08.04:** Визуализация прогресса в виде прогресс-бара

### 3.6. Настройки системы
**FR-09:** Система должна предоставлять настройки
- **FR-09.01:** Выбор специальности студента
- **FR-09.02:** Настройка подключения к БД (SQLite/PostgreSQL)
- **FR-09.03:** Управление источником данных компетенций

## 4. Нефункциональные требования

### 4.1. Требования к производительности
**NFR-01:** Время отклика интерфейса
- Открытие любой вкладки: ≤ 2 секунд
- Загрузка списка целей: ≤ 1 секунды
- Генерация отчета: ≤ 30 секунд для 100 целей

**NFR-02:** Объем данных
- Поддержка до 10 000 целей
- Поддержка до 500 уникальных навыков
- Поддержка до 100 компетенций

### 4.2. Требования к надежности
**NFR-03:** Доступность
- Система должна быть доступна 99% времени работы приложения
- Автоматическое восстановление после сбоев БД

**NFR-04:** Целостность данных
- Проверка корректности вводимых дат
- Валидация обязательных полей
- Автоматическое резервное копирование при изменении структуры БД

### 4.3. Требования к удобству использования
**NFR-05:** Интерфейс пользователя
- Интуитивно понятный интерфейс с вкладками
- Единый стиль оформления всех элементов
- Контекстные подсказки и валидация ввода
- Предпросмотр Markdown разметки в реальном времени

**NFR-06:** Документация
- Встроенная справка по основным функциям
- Подсказки при наведении на элементы интерфейса
- Примеры использования Markdown разметки

### 4.4. Технические требования
**NFR-07:** Совместимость
- Операционные системы: Windows 10+, Linux, macOS
- Версия Python: 3.8+
- Поддержка двух СУБД: SQLite и PostgreSQL

**NFR-08:** Безопасность
- Локальное хранение данных
- Отсутствие сетевых функций (кроме опционального подключения к PostgreSQL)
- Проверка входных данных для предотвращения SQL-инъекций

### 4.5. Требования к сопровождению
**NFR-09:** Модифицируемость
- Модульная архитектура для легкого расширения
- Конфигурация через JSON-файлы
- Возможность добавления новых типов целей без изменения кода

**NFR-10:** Масштабируемость
- Возможность добавления новых модулей
- Поддержка плагинов для экспорта в разные форматы

## 5. Бизнес-правила

### 5.1. Правила валидации
**BR-01:** Правила для целей
- Название цели обязательно для заполнения
- Тип цели должен выбираться из предопределенного списка
- Плановая дата должна быть в формате ГГГГ-ММ-ДД
- Фактическая дата не может быть раньше даты создания цели
- Не более 3 навыков может быть связано с одной целью

**BR-02:** Правила для компетенций
- Уровень компетенции должен быть в диапазоне 0-5
- Компетенции загружаются из файла при первом запуске
- Категория компетенции не обязательна, но рекомендуется

**BR-03:** Правила для достижений
- Достижения присваиваются автоматически при выполнении условий
- Условия проверяются при каждом изменении данных
- Достижение нельзя получить повторно

### 5.2. Правила бизнес-процессов
**BR-04:** Процесс планирования цели
1. Студент определяет образовательную потребность
2. Формулирует цель с указанием типа и сроков
3. Связывает цель с релевантными навыками и компетенциями
4. Устанавливает ожидаемый уровень развития компетенций
5. Сохраняет цель и начинает ее выполнение

**BR-05:** Процесс формирования отчета
1. Сбор статистики по всем целям
2. Анализ развития компетенций
3. Выявление слабых зон
4. Формирование рекомендаций
5. Создание структурированного документа

## 6. Требования к интеграции

### 6.1. Внешние интерфейсы
**IR-01:** Система управления компетенциями
- Загрузка списка компетенций из JSON-файла
- Формат файла: массив объектов с полями "название" и "категория"
- Автоматическое создание файла-примера при первом запуске

**IR-02:** Система баз данных
- Поддержка SQLite для локального использования
- Поддержка PostgreSQL для сетевого использования
- Автоматическое определение доступности PostgreSQL

### 6.2. Интерфейсы данных
**IR-03:** Формат экспорта данных
- Экспорт отчетов в формате DOCX
- Использование библиотеки python-docx для генерации
- Поддержка форматирования текста, таблиц и гиперссылок

**IR-04:** Формат хранения данных
- Реляционная структура с 6 основными таблицами
- Связи между целями, навыками и компетенциями
- История изменений через временные метки

## 7. Ограничения

### 7.1. Технические ограничения
**LC-01:** Архитектурные ограничения
- Desktop-приложение на Python/Tkinter
- Нет веб-интерфейса
- Нет мобильной версии

**LC-02:** Ограничения функциональности
- Нет многопользовательского режима
- Нет ролевой модели (только студент)
- Нет интеграции с системами ВУЗа (Moodle, 1С и т.д.)

### 7.2. Бизнес-ограничения
**LC-03:** Ограничения лицензирования
- Использование open-source библиотек
- Бесплатное распространение для образовательных учреждений
- Открытый исходный код

**LC-04:** Ограничения развертывания
- Требуется установка Python и зависимостей
- Необходимы права на запись в директорию установки
- Для PostgreSQL требуется отдельная установка СУБД

## 8. Словарь терминов

| Термин | Определение |
|--------|-------------|
| Индивидуальный образовательный маршрут | Персонализированный план развития студента, включающий цели, сроки и ожидаемые результаты |
| Компетенция | Способность применять знания, умения и личностные качества для успешной деятельности |
| Навык | Умение выполнять конкретные действия, приобретенное в результате обучения и практики |
| Образовательная цель | Конкретный, измеримый и ограниченный по времени результат, который студент планирует достичь |
| Семестровая цель | Стратегическая цель на учебный семестр, состоящая из нескольких тактических целей |
| Достижение | Виртуальная награда за выполнение определенных условий в системе |
| Markdown | Упрощенный язык разметки для форматирования текста |

## 9. Допущения и зависимости

### 9.1. Допущения
1. Студент обладает базовыми навыками работы с компьютером
2. Образовательное учреждение предоставляет список рекомендуемых компетенций
3. Преподаватели готовы консультировать студентов по постановке целей
4. Имеется доступ к компьютеру для регулярной работы с системой

### 9.2. Зависимости
1. Наличие установленного Python 3.8 или выше
2. Доступ к интернету для установки зависимостей (только первоначально)
3. Для использования PostgreSQL - установленная и настроенная СУБД
4. Наличие текстового редактора, поддерживающего DOCX формат
