# План тестирования системы "Планировщик индивидуального образовательного маршрута"

## 1. Введение

### 1.1. Цель документа
Документ описывает стратегию, объем, подходы, ресурсы и расписание тестирования системы "Планировщик индивидуального образовательного маршрута".

### 1.2. Область действия
Тестирование охватывает все модули системы: управление целями, управление навыками и компетенциями, систему достижений, отчетность и настройки.

### 1.3. Цели тестирования
1. Подтверждение соответствия системы бизнес-требованиям
2. Обеспечение качества и стабильности работы
3. Выявление дефектов на ранних этапах
4. Проверка удобства использования интерфейса
5. Оценка производительности при различных нагрузках

## 2. Стратегия тестирования

### 2.1. Подходы к тестированию
- **Комбинированный подход**: ручное и автоматизированное тестирование
- **Многоуровневое тестирование**: модульное, интеграционное, системное
- **Тестирование на основе рисков**: приоритет на критическую функциональность
- **Итерационный подход**: тестирование после каждой значимой доработки

### 2.2. Типы тестирования
| Тип тестирования | Описание | Инструменты |
|-----------------|----------|-------------|
| Функциональное | Проверка соответствия требованиям | PyTest, ручное тестирование |
| Интерфейсное | Проверка GUI и UX | SikuliX, ручное тестирование |
| Интеграционное | Проверка взаимодействия модулей | PyTest, unittest |
| Нагрузочное | Проверка производительности | Locust, ручное тестирование |
| Безопасности | Проверка уязвимостей | Bandit, ручное тестирование |
| Совместимости | Проверка на разных ОС | VirtualBox, Docker |
| Регрессионное | Проверка после изменений | PyTest, тест-кейсы |

### 2.3. Критерии начала/окончания тестирования
**Критерии начала:**
- Завершена разработка всех модулей
- Готовы тестовые сценарии
- Настроены тестовые среды
- Доступны тестовые данные

**Критерии окончания:**
- Выполнены все запланированные тесты
- Критические дефекты исправлены
- Все блокеры устранены
- Достигнуты целевые показатели качества

3. **Нагрузочный набор**: 1000 целей, 100 навыков, 20 компетенций
4. **Крайние случаи**: пустые данные, некорректные форматы, большие объемы текста

## 7. Детальные тест-кейсы

### 7.1. Модуль управления целями

**TC-001: Создание новой цели**
```
Предусловие: Открыта вкладка "Мои цели"
Шаги:
1. Нажать кнопку "Добавить цель"
2. Заполнить все обязательные поля:
   - Название: "Изучение Python"
   - Тип: "Курс"
   - Статус: "В процессе"
3. Нажать "Сохранить"
Ожидаемый результат: Цель отображается в списке целей
```

**TC-002: Редактирование цели**
```
Предусловие: В списке есть хотя бы одна цель
Шаги:
1. Выбрать цель в списке
2. Нажать кнопку "Редактировать"
3. Изменить статус на "Завершено"
4. Указать фактическую дату
5. Нажать "Сохранить"
Ожидаемый результат: Изменения сохранены, цель обновлена в списке
```

**TC-003: Удаление цели**
```
Предусловие: В списке есть хотя бы одна цель
Шаги:
1. Выбрать цель в списке
2. Нажать кнопку "Удалить"
3. Подтвердить удаление
Ожидаемый результат: Цель удалена из списка
```

**TC-004: Валидация дат**
```
Предусловие: Открыто окно добавления цели
Шаги:
1. Ввести некорректную дату: "2023-13-45"
2. Попытаться сохранить
Ожидаемый результат: Сообщение об ошибке, сохранение невозможно
```

**TC-005: Добавление навыков к цели**
```
Предусловие: Открыто окно добавления/редактирования цели
Шаги:
1. В поле навыков начать вводить "Програм"
2. Выбрать "Программирование" из автодополнения
3. Добавить еще 2 навыка
4. Сохранить цель
Ожидаемый результат: Навыки связаны с целью, отображаются в статистике
```

import pytest
from app import EducationalPlanner

class TestGoalManagement:
    def test_create_goal(self):
        """Тест создания цели"""
        planner = EducationalPlanner()
        goal_id = planner.create_goal(
            name="Test Goal",
            goal_type="Course",
            status="In Progress"
        )
        assert goal_id is not None
        
    def test_goal_validation(self):
        """Тест валидации дат"""
        planner = EducationalPlanner()

            planner.create_goal(
                name="Invalid Date",

            )
