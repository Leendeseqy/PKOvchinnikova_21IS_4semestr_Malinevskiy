### 1.2. Основание для разработки
Разработка учебного проекта в рамках дисциплины "Программная инженерия" для демонстрации навыков создания клиент-серверных приложений.

Создание безопасной платформы для внутренней коммуникации в локальных сетях образовательных учреждений без необходимости подключения к интернету.

### 2.3. Паттерны проектирования

#### 2.3.1. Клиентская часть
1. **MVC (Model-View-Controller):**
   - Model: `Message`, `User` классы
   - View: PyQt5 виджеты (ChatWidget, MainWindow)
   - Controller: Обработчики событий и бизнес-логика

2. **Observer Pattern:**
   - WebSocket уведомления через сигналы PyQt5
   - Обновление UI при изменении данных

3. **Singleton Pattern:**
   - Конфигурация приложения
   - WebSocket клиент

#### 2.3.2. Серверная часть
1. **Repository Pattern:**
   - `UserModel`, `MessageModel` классы
   - Абстракция доступа к данным

2. **Dependency Injection:**
   - FastAPI Depends для внедрения зависимостей
   - Автоматическое внедрение текущего пользователя

3. **Factory Method:**
   - Создание JWT токенов
   - Создание объектов сообщений

### 2.4. Структура базы данных

```sql
-- Основные таблицы
USERS (id, username, password_hash, is_online, last_seen, status, is_admin, created_at)
MESSAGES (id, sender_id, receiver_id, content, message_type, file_data, is_read, timestamp)

-- Связи
FOREIGN KEY (sender_id) REFERENCES USERS(id)
FOREIGN KEY (receiver_id) REFERENCES USERS(id)

-- Индексы для производительности
CREATE INDEX idx_messages_sender ON MESSAGES(sender_id);
CREATE INDEX idx_messages_receiver ON MESSAGES(receiver_id);
CREATE INDEX idx_messages_timestamp ON MESSAGES(timestamp);
CREATE INDEX idx_users_username ON USERS(username);
```

### 2.5. Схема взаимодействия компонентов

```
Последовательность отправки сообщения:
1. [Клиент] User → ChatWidget.send_message()
2. [Сеть] HTTP POST → /api/messages
3. [Сервер] FastAPI → messages.py → MessageModel.create_message()
4. [БД] INSERT INTO messages
5. [Сервер] WebSocket → manager.send_personal_message()
6. [Клиент] WebSocketClient → handle_websocket_message()
7. [UI] ChatWidget → add_message_to_display()
```

### 5.3. Интерфейс базы данных

#### 5.3.1. Схема таблицы users
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    is_online BOOLEAN DEFAULT FALSE,
    last_seen DATETIME,
    status TEXT DEFAULT 'offline',
    is_admin BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 5.3.2. Схема таблицы messages
```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sender_id INTEGER NOT NULL,
    receiver_id INTEGER NOT NULL,
    content TEXT NOT NULL,
    message_type TEXT DEFAULT 'text',
    file_data TEXT,
    is_read BOOLEAN DEFAULT FALSE,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (sender_id) REFERENCES users (id),
    FOREIGN KEY (receiver_id) REFERENCES users (id)
);
```

### 5.5. Протоколы передачи данных

#### 5.5.1. HTTP/HTTPS протокол
- **Версия:** HTTP/1.1

## 6. Требования к безопасности

### 6.1. Аутентификация и авторизация
1. **JWT токены** с временем жизни 24 часа
2. **Хеширование паролей** алгоритмом bcrypt
3. **Проверка прав** на уровне endpoints
4. **Автоматический логаут** при истечении токена

### 6.2. Защита данных
1. **SQL инъекции:** Использование параметризованных запросов
2. **XSS атаки:** Экранирование HTML в сообщениях
3. **CSRF:** Использование JWT вместо сессий
4. **Перехват трафика:** Рекомендация использовать HTTPS в production

### 6.3. Конфиденциальность
1. **Пароли** никогда не передаются в открытом виде
2. **Файлы** передаются в base64 кодировке
3. **Личные данные** не передаются третьим лицам
4. **Логи** не содержат чувствительной информации
